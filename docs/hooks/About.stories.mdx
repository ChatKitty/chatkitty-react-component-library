# <u>Hooks</u>

Hooks are a new addition in React 16.8. They let you use state and other React features without writing a class.

The [motivation](https://reactjs.org/docs/hooks-intro.html) behind hooks was to make it easier to [1] reuse stateful logic between components, [2] make complex components easier to understand, and [3] avoid class components.

## ChatKitty Custom Hooks

ChatKitty's custom hooks exist to provide a helpful React-friendly abstraction over ChatKitty's JS SDK.

These hooks are able to:

- Call the correct SDK methods given a ChatKitty client instance and return the associated `resource`.
- Provide request loading states through the `isLoading` property.
- Provide request errors through the `error` property.
- Fetch or refetch data through the `makeRequest` property.

```typescript
const App = () => {
  // Let's use a hook to start a ChatKitty guest session!
  const { isLoading } = useGuestSession(client, username);

  return isLoading ? <Spinner /> : <Chat />;
};
```
